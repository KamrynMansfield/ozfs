Zoning Accessibility Analysis Project

Intro to website Purpose Background Methodology - Data Standard - Create Software - Perform Analysis

The purpose of this research is to analyze how much housing current zoning laws will accommodate by creating software to perform this analysis along many municipalities in an area. A main focus of this project will be the creation of software that makes it possible to repeat the analyses in other cities.

This website is meant to provide clear documentation of our research process. The sections on this page will describe the purpose, background, and methodology for the research. The other pages on the website help to document specific aspects of the project.

Background

Cities are getting denser and housing is getting more expensive. Many people are looking for ways to provide more housing.

There has been research done about affects of changing zoning codes on housing availability, but there has been less done on the housing capacity that existing zoning codes allow. We want to see how much housing the current zoning codes would accommodate if they were left as they are.

The National Zoning Atlas is an effort to record zoning codes as they are, but it mostly focuses on the the big picture to visualize where certain land uses are permitted. This may help cities and developers see where to start looking to build or where changes may need to occur, but it isn’t designed to take a building design and show where it is allowed. The National Zoning Atlas is also not designed to predict the housing capacity of areas.

Methodology

Since we are more focused on creating software to perform these analyses, we didn’t start with a specific area of analysis in mind. Practically any area would work as long as it has zoning codes and corresponding parcel and road data for the area. We chose Dallas - Fort Worth (DFW) as our area of analysis because Lauren Fischer was kind enough to share the data she already gathered with us. She led the work in putting the DFW zoning codes into the National Zoning Atlas format, and she gave us access to the data. It is around 200 municipalities and is already in a standard, computer-readable format.

After specifying the data source and study area, our methodology consists of 3 main steps: Creating a data standard, creating software, and performing the analysis

Data Standard

We want to have the zoning data in a format that allows little room for interpretation of the zoning code. Most zoning codes contain similar information (minimum setbacks, minimum floor area, maximum height, etc.), but they can be so different in the way they express this information. For example, some cities will have a fixed maximum height that can’t be exceeded, while others have a maximum height that can be adjusted if the setback is adjusted.

While the data we have is already in a computer-readable format created using the Zoning Atlas methodology, it doesn’t accomplish all of our goals. The lack of detail in some areas of the Zoning Atlas format means that transferring the data to the format may have required some interpretation.

The exact information may not be necessary for creating software that can perform different analyses with the zoning codes, but we wanted to format the data in a way that was a bit more detailed and adaptable to the many zoning codes out there. For this reason, we have created our own feed specification: the Open-Zoning Feed Specification (OZFS).

We plan to use the DFW data found in the National Zoning Atlas format and put it into our OZFS format before running the analysis. Future data that is translated to the OZFS format directly from city zoning codes will most likely contain more data than the data used in this study.

Software

The analysis will be completed using an R (and potentially Python) package called tidyzoning. For more information about the tidyzoning package, see the documentation found on the tidyzoning page of this website.

The list below describes some of main tasks we hope the package will accomplish.

  • Take a building plan and find out where the zoning codes will allow it.
  • Take a parcel and find out what kind of building we can build on it.

In order to complete these tasks, it will be important to have a uniform way of describing buildings, parcels, and zoning codes. The tidyzoning documentation will describe the data classes made to represent these three things.

  1. tidybuilding: this data class stores information to represent a building
  2. tidyparcel: this data class stores information to represent a parcel
  3. tidyzoning: this data class stores information to represent the municipality’s zoning codes.

Analysis

Once the above package is finished, we will generate tidybuilding objects representing each of several building types (assume 2-bedroom units):

  1. A two-family home
  2. A three-family home
  3. A four-family home
  4. A six-family home
  5. A 12-family home

Then we will identify all the parcels in the study area that can accommodate each of these types under the following conditions:

  1. All parcels are considered to be available
  2. All unoccupied parcels are considered to be available
  3. All vacant parcels are considered to be available
  4. All parcels that have been sold in the past 12 months are considered to be available (representing the set of parcels that could be sold in a typical year)

We also hope to calculate average accessibility to “attractions” with and without a car under existing conditions and under each of the 20 scenarios above.
